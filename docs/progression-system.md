# 학생 레벨 · 경험치 · 캐릭터 육성 시스템

> 이후 **모든 시뮬레이션·미니게임**은 이 문서를 따른다.  
> 코드 단일 출처: [`lib/xp.ts`](../lib/xp.ts), [`lib/progression.ts`](../lib/progression.ts)

---

## 1. 한 줄 요약

학생은 과제를 클리어할수록 **XP → 레벨업 → 파이 외형 교체 → 장비 해금 → 동료 합류**를 얻는다.  
한 판의 점수가 곧 경험치이며, **만점 ≈ 1000점**을 목표로 콘텐츠를 설계한다.

---

## 2. 수치 프레임 (고정)

| 항목 | 값 |
|------|-----|
| 한 판 점수 | **0 ~ 1000** (정수) |
| 점수 → XP | **1:1** (`scoreToXp`가 클램프) |
| 만렙 | **Lv.100** |
| 만렙 누적 XP | **500,000** (만점 약 500판) |
| 곡선 지수 | **2.25** (초반 빠름 → 후반 완만) |

### 누적 XP 공식

```
cumulativeXp(L) = floor( 500_000 * ((L - 1) / 99) ^ 2.25 )
```

- Lv.1 = 0 XP  
- Lv.100 = 500,000 XP  

SQL: `pm_level_from_xp` / `pm_cumulative_xp_for_level` (동일 식).

### 샘플 임계값 (대략)

| 레벨 | 누적 XP | 비고 |
|------|---------|------|
| 2 | 매우 적음 | 첫 판으로도 상승 |
| 5 | ~수백 | 초원 해금 |
| 10 | — | 폼·아이템 연쇄 |
| 15 | — | 언덕 해금 |
| 30 | — | 별빛 해금 |
| 50 | 중반 | |
| 100 | 500,000 | 만렙 |

정확한 값은 `sampleLevelThresholds()` 참고.

## 3. 시뮬레이션 · 게임 제작 규칙 (필수)

1. **한 판 만점은 대략 1000점**.  
2. 부분 점수 허용 (0~1000). 1000 초과 금지.  
3. 클리어 후 서버에서만 XP 부여.

```ts
import { awardStudentXp } from "@/app/adventure/actions";
await awardStudentXp({ gameKey: "sim-fractions-1", score: earnedScore });
```

4. `gameKey`는 콘텐츠 고유 문자열.  
5. 같은 판 반복 클리어 시에도 XP 누적 (연습 장려).

| 성과 | 권장 점수대 |
|------|-------------|
| 대충 참여 | 100–300 |
| 보통 클리어 | 400–700 |
| 잘함 | 750–900 |
| 거의 만점 | 950–1000 |
